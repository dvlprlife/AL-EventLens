# Change Log

All notable changes to the AL EventLens extension will be documented in this file. This project adheres to [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and uses [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- AL source parser (`src/al/parser.ts`) recognizes publisher (`[IntegrationEvent]`, `[BusinessEvent]`) and subscriber (`[EventSubscriber]`, both pre-BC22 and BC22+ syntaxes) declarations across all AL object kinds, returning typed `Publisher` and `Subscriber` records with `vscode.Location` pointing at the procedure name.
- Trigger-event synthesis (`src/al/triggers.ts`) produces 10 virtual publishers per Table object (`OnBeforeInsertEvent`, `OnAfterInsertEvent`, `OnBeforeModifyEvent`, `OnAfterModifyEvent`, `OnBeforeDeleteEvent`, `OnAfterDeleteEvent`, `OnBeforeRenameEvent`, `OnAfterRenameEvent`, `OnBeforeValidateEvent`, `OnAfterValidateEvent`) and 8 per Page object (`OnOpenPageEvent`, `OnClosePageEvent`, `OnQueryClosePageEvent`, `OnInsertRecordEvent`, `OnModifyRecordEvent`, `OnDeleteRecordEvent`, `OnNewRecordEvent`, `OnAfterGetCurrRecordEvent`). The indexer will gate this on the `alEventLens.includeTriggerEvents` setting.
- `.app` package reader (`src/symbols/appReader.ts`) reads BC application packages via `vscode.workspace.fs` (web-compatible), validates the 40-byte NAVX header, decompresses the embedded PKZIP with JSZip, and extracts the `appId`, `version`, `SymbolReference.json` body, and bundled AL source files under `src/**`. `.NEA` runtime packages are detected by extension and rejected with a clear error rather than a silent failure.
- Legacy `SymbolReference.json` schema parser (`src/symbols/schemaFlat.ts`) extracts publisher events (`IntegrationEvent`, `BusinessEvent`) from the pre-BC24 flat schema for Codeunits, Tables, Pages, Reports, Queries, XmlPorts, and Interfaces. Accepts either a JSON string (with optional UTF-8 BOM, common in Microsoft `.app` packages) or an already-parsed object.
- Nested-namespaces `SymbolReference.json` schema parser (`src/symbols/schemaNamespaces.ts`) handles the BC24+ schema where object-kind arrays live recursively inside `Namespaces[]` entries. Walks both root-level arrays and nested namespaces at any depth. Reuses `extractPublishersFromContainer` from `schemaFlat.ts` so the per-object/per-method walk isn't duplicated. With this, `detect.ts` routes both schemas correctly: `Namespaces` key present → nested parser, otherwise → flat parser. The full `.app` ingestion pipeline (`readApp` → `parseSymbolReference`) is now operational; the indexer just needs to wire it into a workspace scan.
- Subscriber-to-publisher resolver (`src/index/resolver.ts`) sets the `resolved` flag on each subscriber by matching `(kind, name, eventName)` case-insensitively against the workspace publisher set. Trigger publishers participate equally with `[IntegrationEvent]` / `[BusinessEvent]` publishers; `Subscriber.target.appId` is ignored so cross-app resolution works. Pure function over already-parsed `Publisher[]` and `Subscriber[]` — no I/O, no `vscode` runtime imports — ready to compose into the workspace indexer.
- Workspace event indexer (`src/index/indexer.ts`) composes the AL parser, trigger synthesizer, `.app` reader, schema parsers, and subscriber resolver into a single `buildIndex(context)` call. Walks `**/*.al` under workspace folders and `**/.alpackages/*.app`, gates package scanning on `alEventLens.scanAlpackages` and trigger synthesis on `alEventLens.includeTriggerEvents`, tolerates a corrupted or `.NEA`-rejected `.app` per package via `console.warn` without aborting the run, and returns a fully resolved `EventIndex`. `extension.ts` invokes it during `activate()` and logs publisher/subscriber counts for verification — no panel, tree, or CodeLens wiring yet.
- Persistent `.app` symbol cache (`src/index/cache.ts`) stores parsed publisher lists under `extensionContext.globalStorageUri/symbols/{appId}__{version}__{mtime}.json` via `vscode.workspace.fs`, gated by `alEventLens.cache.enabled`. The indexer checks the cache before re-parsing each `SymbolReference.json` and cleans up older entries for the same `appId` on version bumps, making second-and-later workspace opens near-instant for the `.app` ingestion path.
- Long-lived event index store (`src/index/store.ts`) — the `EventIndexStore` class wraps the current `EventIndex` behind a `vscode.EventEmitter`-backed `onDidChange`, with `set` (full replace, used by the activation-time `buildIndex`) and `updateFile` (per-URI replace that re-runs `resolveSubscribers` so cross-file links stay correct). Trigger publishers carry an optional `sourceUri` so the store can identify and replace them on a subsequent save instead of accumulating duplicates. Pushed into `context.subscriptions` from `extension.ts` so disposal is automatic on extension shutdown.
- Save watcher (`src/index/watcher.ts`) — `registerSaveWatcher(context, store)` subscribes to `vscode.workspace.onDidSaveTextDocument`, filters AL documents, re-parses with `parseAl`, optionally synthesizes Table/Page trigger publishers (gated by `alEventLens.includeTriggerEvents`), and routes the result into `EventIndexStore.updateFile`. Gated by `alEventLens.indexOnSave`. The inner `handleSave` helper is exported so tests can drive it directly without stubbing the workspace event surface. The trigger-owner extraction helper (`collectTriggerOwners`) moved from `src/index/indexer.ts` to `src/al/triggers.ts` so both the full-pass indexer and the incremental save watcher can share it.
- Activity-bar TreeView (`src/ui/treeView.ts`) — registers `alEventLensView` as a `vscode.TreeView` populated from `EventIndexStore`. Publishers are grouped by source `appId` (`(workspace)` bucket first, then alphabetical), each leaf labeled `Kind::"Name" · EventName · (N)` where `N` is the live subscriber count from the store's resolver-keyed match. Trigger publishers participate alongside `[IntegrationEvent]` / `[BusinessEvent]`. Subscribes to `store.onDidChange` so the tree refreshes whenever `buildIndex` completes or the save watcher updates a file. Each leaf carries a `command: alEventLens.revealPublisher` payload — the panel PR will wire the handler.
- Subscriber-count CodeLens (`src/ui/codelens.ts`) — `registerCodeLens(context, store)` registers a `vscode.CodeLensProvider` for the `al` language that draws a `"N subscribers"` lens above each `[IntegrationEvent]` and `[BusinessEvent]` declaration in the active document, with live count computed from `EventIndexStore`. Pluralization: `1 subscriber` / `0 subscribers` / `N subscribers`. Clicking the lens fires `alEventLens.revealPublisher` with the publisher ref. Trigger publishers carry no source location and so get no lens. Gated by `alEventLens.codeLens.enabled`; the provider re-fires `onDidChangeCodeLenses` when the store mutates and when the setting toggles, so toggling takes effect without a reload. `extension.ts` now threads the store into the registration call.
- Mermaid export (`src/ui/mermaid.ts`, `src/index/match.ts`) — `renderMermaid(publisher, subscribers)` produces a deterministic `graph LR` diagram with one publisher node and `S1`..`SN` subscriber nodes sorted by `(owner.appId, owner.kind, owner.name, line)`; resolved subscribers use `-->`, unresolved use `-.->`; labels escape `"` → `&quot;` and newlines → `<br/>`. `alEventLens.exportMermaid` now reads the panel selection (or accepts an optional `Publisher` arg), copies the diagram to the clipboard via `vscode.env.clipboard`, and shows a confirmation toast. The panel now publishes its current selection back to the extension via a `selectionChanged` message, and `getSelectedPublisher()` is exported for command-palette invocations. The previously-duplicated subscriber-match logic in `resolver.ts`, `codelens.ts`, and `treeView.ts` is consolidated in the new `src/index/match.ts` (`publisherKey`, `subscriberKey`, `findSubscribersFor`, `countSubscribersByPublisherKey`); the webview keeps its own JS-encoded copy because it cannot import TS modules.
- Searchable webview panel (`src/ui/panel.ts`, `src/ui/panelHtml.ts`) — `openPanel(context, store)` creates a singleton `vscode.WebviewPanel` ("AL EventLens", `ViewColumn.Beside`) with a two-pane layout: a 30%-wide publisher list with case-insensitive substring search on the left, and a subscriber-detail pane on the right that lists each subscriber with `✓` / `⚠` resolved-status badges, file path, and line number. The panel subscribes to `EventIndexStore.onDidChange` so it re-paints on every workspace re-index and every incremental save. Self-contained HTML with a CSP-nonce-tagged inline script — no external resources, works identically on desktop and `vscode.dev`. The `alEventLens.refresh`, `alEventLens.revealPublisher`, and `alEventLens.gotoSubscriber` commands now have real implementations: refresh re-runs `buildIndex` into the store, revealPublisher opens the panel and posts a `select` message, and gotoSubscriber opens the target document at the subscriber's location. `extension.ts` constructs the store before registering the commands so the handlers can close over it.
- Resizable webview panel divider (`src/ui/panelHtml.ts`) — the previously-static 30%/70% split between the publisher list and the subscriber detail is now a 4-px draggable separator using `--vscode-panel-border` (tinted with `--vscode-focusBorder` on hover and during drag). Pointer events update a `--left-width` CSS variable, clamped to a 150-px minimum on the left and a 200-px minimum on the right. The chosen width is persisted via the webview's `vscode.setState` / `getState` so it survives panel hide/reveal (`retainContextWhenHidden` is already on); double-clicking the divider clears the override and snaps back to the 30% default.
- Per-row subscriber count in the webview panel's publisher list (`src/ui/panelHtml.ts`) — each list item now ends with ` · (N)` (the same format used by the tree leaves), where `N` is the live subscriber count computed via the existing webview-side `findSubscribersFor` helper. Resolved and unresolved subscribers are both included, consistent with the tree and the CodeLens.
- Hover tooltips on webview panel rows (`src/ui/panelHtml.ts`) — publisher rows show `"<Kind> event · N subscriber(s)\n<appId or (workspace)>"`; resolved subscriber rows show `"<kind>::\"<name>\"\n<full path>:<line>\nClick to jump to source"`; unresolved subscriber rows show the owner ref plus `"Unresolved — target app missing from .alpackages"`. Uses native HTML `title` attributes — no JS, no extra DOM, themed correctly inside the webview.
- Aggregate counts on TreeView intermediate rows (`src/ui/treeView.ts`) — `KindNode` and `ObjectNode` `TreeItem.description` fields show `(events / subscribers)` — the number of publisher events under that subtree plus the sum of subscriber counts across those events — so the tree can be skimmed without expanding everything and a busy subtree is obvious at a glance. `EventNode` leaves keep their existing `· (N)` subscriber count in the label; `AppNode` description continues to carry the `appPublisher` vendor name.
- Codicon icons on every TreeView node level (`src/ui/treeView.ts`) — `vscode.ThemeIcon` `iconPath` set on each row: `package` on app rows, a per-kind codicon on `KindNode` rows (`symbol-class` for codeunit, `symbol-struct` for table/tableextension, `window` for page/pageextension, `notebook` for report/reportextension, `search` for query, `file-code` for xmlport, `symbol-enum` for enum/enumextension, `lock` for permissionset, `symbol-interface` for interface), `symbol-file` on object rows, `symbol-event` on event leaves, and `info` on the empty-state placeholder. All icons tint with the current VS Code theme automatically — no extra assets in the bundle.

### Changed

- Dual-platform bundle: `package.json` now declares both `main` (Node bundle at `dist/extension.js`) and `browser` (Web Worker bundle at `dist/web/extension.js`). VS Code Desktop loads the Node bundle; `vscode.dev` and `github.dev` continue to load the browser bundle. Source-level web-discipline is preserved — the browser bundle is built from the same source and would fail to build if anything used Node-only APIs.
- Publishers tree (`src/ui/treeView.ts`) now shows the friendly app `Name` from each `.app`'s `NavxManifest.xml` as the row label, the `Publisher` (vendor) as the row description, and the full `appId` GUID in the tooltip. Buckets without a matching `appMeta` entry — older `.app` packages whose manifest omits `Name` — fall back to the raw GUID. Sort order is now case-insensitive on the displayed label (`(workspace)` still first). To support this, `parseManifest` in `src/symbols/appReader.ts` now extracts the optional `Name` / `Publisher` attributes (no throw when missing), `EventIndex` carries a `ReadonlyMap<string, AppMeta>` propagated through `EventIndexStore`, and the per-`.app` cache schema is bumped to v2 (`{ schemaVersion, publishers, name?, appPublisher? }`); pre-existing v1 cache entries are silently treated as misses and re-parsed. The new field is named `appPublisher` throughout — not `publisher` — to avoid colliding with AL EventLens's domain concept of an *event publisher*.
- TreeView hierarchy (`src/ui/treeView.ts`) now drills four levels — source app → AL object kind → object → event — instead of one flat publisher row per event under the app bucket. `KindNode` rows (`Codeunit`, `Table`, `XmlPort`, ...) sort alphabetically by display label; `ObjectNode` rows sort case-insensitively by `owner.name`; `EventNode` leaves sort by event name and keep the existing `· (N)` subscriber count plus the `alEventLens.revealPublisher` click command. Trigger publishers participate alongside `[IntegrationEvent]` / `[BusinessEvent]` publishers under their respective kind bucket. The leaf-label format changed from `Kind::"Name" · EventName · (N)` to `EventName · (N)` since `Kind` and `Name` now live on the parent rows.

### Fixed

- Activity bar icon (`package.json`, `images/activity-bar.svg`) was rendering as a grey placeholder because the contributed `viewsContainers.activitybar` entry pointed at `images/icon.png`, and VS Code's activity bar refuses to render colored PNGs — it requires a monochrome SVG that uses `currentColor` so it can re-tint per theme. Added a separate `images/activity-bar.svg` (magnifying glass + 3 connected event nodes, matching the marketplace PNG's concept in monochrome) and pointed the activity-bar `icon` field at it; the top-level marketplace `icon` field still uses `images/icon.png`.
- Duplicate publishers from `.alpackages/*.app` packages that ship bundled `src/*.al` source (`src/index/indexer.ts`) — every event under any such package (Microsoft BaseApp, Business Foundation, Test Runner, etc.) was appearing twice in the panel list, tree, and CodeLens counts. The indexer was pushing publishers from both `parseSymbolReference` (the authoritative source per CLAUDE.md) AND from `parseAl` over the bundled `src/*.al` files. The bundled-source pass now contributes subscribers and trigger owners only; publishers come exclusively from `SymbolReference.json`. New regression test asserts an event declared in both surfaces appears exactly once.

## [0.1.0] - 2026-05-14

Initial public release.

### Added

- Workspace-wide indexing of event publishers (`[IntegrationEvent]`, `[BusinessEvent]`) and subscribers (`[EventSubscriber]`) from `.al` files. Both pre-BC22 (`'Codeunit Name'`, `'OnEvent'`) and BC22+ (`Codeunit::"Name"`, `OnEvent`) subscriber syntaxes are recognized.
- Indexing of publishers from `.alpackages/*.app` packages via `SymbolReference.json`, supporting both the legacy flat schema and the BC 24+ nested `Namespaces[]` schema. Encrypted `.NEA` runtime packages are detected and skipped with a clear diagnostic.
- Trigger-event synthesis: virtual publishers for `OnAfterDeleteEvent`, `OnBeforeValidateEvent`, etc. on Table and Page objects, gated by `alEventLens.includeTriggerEvents`.
- Subscriber → publisher link resolution with **resolved** / **unresolved** status per subscriber.
- Webview panel — searchable publisher list with subscriber-count badges and a subscriber-detail view; jump-to-source on click.
- Activity-bar view (`AL EventLens`) listing publishers grouped by source app.
- CodeLens above `[IntegrationEvent]` and `[BusinessEvent]` declarations showing the live subscriber count, gated by `alEventLens.codeLens.enabled`.
- Mermaid export command copies the current publisher's subscriber graph to the clipboard as a Mermaid diagram.
- Incremental re-index on file save, gated by `alEventLens.indexOnSave`.
- Persistent cache in `extensionContext.globalStorageUri` keyed by `(appId, version, mtime)`, gated by `alEventLens.cache.enabled`.
- Five `alEventLens.*` settings governing cache, save-watching, package scanning, trigger synthesis, and CodeLens display.
- VS Code Web compatibility — all file access uses `vscode.workspace.fs` and `.app` decompression uses JSZip, so the extension runs on `vscode.dev` and `github.dev`.
